Principal Component Analysis¶

In some cases, one-hot encoding and other feature engineering can result in an explosion of features.

The increase in the number of features can pose problems with computational tractability and with accuracy (more features is not always better).

PCA provides us a way to reduce the number of features (dimensions) without losing too much data:

Let's see how accuracy changes with different #s of dimensions for wine in a decision tree¶

In [18]:
from sklearn.metrics import accuracy_score
from sklearn.model_selection import train_test_split
from sklearn.tree import DecisionTreeClassifier
In [19]:
dims = np.r_[1: len(wine.feature_names) + 1]
accuracies = []

for dim in dims:
    pca = PCA(n_components=dim)
    X = pca.fit_transform(wine.data)
    X_train, X_test, y_train, y_test = train_test_split(X, wine.target, random_state=1)
    model = DecisionTreeClassifier()
    model.fit(X_train, y_train)
    accuracies.append(accuracy_score(y_test, model.predict(X_test)))
In [20]:
plt.plot(dims, accuracies)

Scaling our inputs¶

It turns out that since PCA is a geometric approach, we can get much better results if we scale our data using a StandardScaler:

In [21]:
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
scaled_data = scaler.fit_transform(wine.data)
In [22]:
dims = np.r_[1: len(wine.feature_names) + 1]
accuracies = []

for dim in dims:
    pca = PCA(n_components=dim)
    X = pca.fit_transform(scaled_data)
    X_train, X_test, y_train, y_test = train_test_split(X, wine.target, random_state=1)
    model = DecisionTreeClassifier()
    model.fit(X_train, y_train)
    accuracies.append(accuracy_score(y_test, model.predict(X_test)))

plt.plot(dims, accuracies);

Trade-offs in Machine Learning¶

... aka "What's the best model?"

Major trade-off #1: Bias/Variance¶

ML Models have three sources of error:

  • Bias is error introduced because the model is too inflexible (e.g. using a linear model for a nonlinear process)
  • Variance is the error introduced because the model is too sensitive to random variations in the input data
  • Irreducible error is error introduced because we don't have the right features (or the causes of variation are uknown)

from sklearn.linear_model import LinearRegression

np.random.seed(1)

X = xs[:,np.newaxis]
m0 = LinearRegression()
m0.fit(X, ys)
m1 = SVR(C=10)

Major trade-off #2: Interpretability vs power / accuracy¶

The most flexible/powerful models (gradient boosting trees, SVM, neural networks, etc.) are also the most difficult to explain

Explanatory power can motivate human decision-makers, so often a weaker model is a "better" solution because it's easy to understand

Sometimes the true objective of your model is to "make sure I don't look foolish."

Major trade-off #3: Power / accuracy vs performance constraints¶

The most powerful model ...

... that needs to be deployed to a mobile phone

... and requires 2 TB of RAM for inference
